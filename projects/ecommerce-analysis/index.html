E-Commerce Data Analysis

End-to-end analytics project featuring database design, Python ETL, and interactive dashboards — all packaged cleanly for the web.

1) Project Overview

This project walks through a full analytics lifecycle on an e-commerce dataset — from raw data to decision-ready dashboards. The flow you’ll see below mirrors a real-world stack:

  • Ingest: Raw CSVs imported into MySQL (normalized tables).
  • Clean & Transform: Data pulled into Python (Pandas / SQLAlchemy) for deduping, type fixes, joins, derived metrics, and validation.
  • Store: Cleaned data written back to MySQL in a single, analysis-ready table.
  • Visualize: Connected to Power BI to build KPI views & deep-dive dashboards.

Focus areas include sales trends, customer behavior, product performance, and payment patterns.

2) Data Pipeline

  • Source → MySQL: Import raw files to customers, orders, order_items, payments, products.
  • Python Cleanup: Null handling, type casting, outlier trimming, feature columns (e.g., order value, delivery days).
  • Warehouse Table: Persist a consolidated, query-friendly table back to MySQL.
  • BI Layer: Model in Power BI for slicers, DAX measures, and visuals.

5) Key Insights

  • Sales peak during holidays with noticeable Q4 spikes.
  • Top 10 customers contribute to ~40% of revenue (long tail beyond that).
  • Credit card dominates payments (~75% of transactions).
  • Avg. delivery ≈ 12 days; delays cluster in a few regions — clear ops opportunity.
