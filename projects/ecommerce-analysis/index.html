The E-commerce Sales Analysis project aimed to convert raw transactional data into meaningful business insights. The project followed a structured end-to-end data pipeline, ensuring data quality, consistency, and usability for decision-making.

The process began with importing raw datasets into MySQL, where the database schema was designed to handle orders, payments, customers, and products efficiently. The imported data was then fetched into Python (Jupyter Notebook) for extensive cleaning and preprocessing. After removing duplicates, handling missing values, and ensuring type consistency, the cleaned data was reloaded into MySQL as a reliable single source of truth.

With the structured database in place, SQL queries were developed to calculate KPIs such as revenue trends, customer retention, delivery performance, and category-level insights. The clean dataset was then connected to Power BI, where interactive dashboards provided a clear view of patterns such as seasonal demand, top-selling products, and contributions from repeat customers.

This workflow highlights expertise across data engineering, analysis, and visualization: starting from raw data ingestion in MySQL, through Python-based data cleaning, to SQL-driven transformations, and finally insight-rich dashboards in Power BI. The project demonstrates how a well-designed pipeline can turn raw data into actionable strategies for E-commerce growth.
